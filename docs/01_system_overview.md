# 01. システム全体仕様

## 1. 目的

M5Crawler は、M5Stack 系デバイスを使った無線操縦クローラロボットです。
コントローラ側でジョイスティック入力を読み取り、ESP-NOW で車体側へ送信します。
車体側は受信した X/Y 操作量から左右履帯用サーボ速度を計算し、4 個のシリアルサーボを
ホイールモードで駆動します。

## 2. ファームウェア構成

| 役割 | PlatformIO 環境 | メインソース | 主な責務 |
| --- | --- | --- | --- |
| コントローラ（送信機） | `m5stick-c-plus2` | `src/controller_main.cpp` | MiniJoyC 読み取り、速度上限切替、ESP-NOW 送信、状態表示 |
| クローラ（車体） | `m5stack-stamps3` | `src/crawler_main.cpp` | ESP-NOW 受信、差動駆動計算、シリアルサーボ制御、Avatar 表示 |
| Stack-chan | `m5stack-cores3-stackchan` | `src/stackchan_main.cpp` | クローラからの BodyCommand 受信、体の向き制御 |

## 4. 操作データの流れ

1. コントローラ起動時に MiniJoyC のニュートラル位置を 10 サンプル平均で取得する。
2. ループ内で MiniJoyC の 8bit 位置値を読み取る。
3. 起動時オフセットを差し引き、`-128..127` に制限する。
4. `x` は `-100..100`、`y` は `100..-100` へマッピングする。
   - `y` は符号方向を反転している。
5. 絶対値 5 未満をデッドゾーンとして 0 にする。
6. A ボタンで最大速度 `maxSpd` を `1000..6000` の範囲で段階切替する。
7. 100ms ごとに `ControlData` を ESP-NOW ブロードキャストで送信する。
8. クローラ側は `ControlData` を受信し、最終受信時刻を更新する。
9. 1 秒以上受信が途絶えた場合、フェイルセーフとして `joyX = 0`, `joyY = 0` にする。
10. `joyX`, `joyY` から左右速度を計算してサーボへ出力する。

## 5. 主要パラメータ

| 項目 | 値 | 定義箇所 | 備考 |
| --- | ---: | --- | --- |
| ESP-NOW チャンネル | 1 | 両ファームウェア | `esp_wifi_set_channel(1, ...)` |
| 送信方式 | MAC 指定 / 未設定時ブロードキャスト | コントローラ | `include/ControlProtocol.h` |
| 送信周期 | 100ms | `controller_main.cpp` | `millis() - lastSendTime > 100` |
| 受信停止フェイルセーフ | 1000ms | `crawler_main.cpp` | 操作量をゼロ化 |
| Avatar 表示更新周期 | 200ms | `crawler_main.cpp` | 表情のみ更新 |
| サーボ UART | Serial1, 1Mbps, 8N1 | `crawler_main.cpp` | RX=2, TX=1 |
| サーボタイムアウト | 10 | `crawler_main.cpp` | `sts.IOTimeOut = 10` |
| 初期最大速度 | 2000 | `ControlProtocol.h` | コントローラ値が受信側へ反映 |
| 最大速度切替 | +1000 / 6000 超過で 1000 | コントローラ | A ボタン |
| ジョイスティックデッドゾーン | ±5 未満 | コントローラ | `x`, `y` とも |

## 6. 用語

| 用語 | 意味 |
| --- | --- |
| コントローラ | 操作用デバイス。M5StickC Plus2 + MiniJoyC Hat。 |
| クローラ | 車体側デバイス。StampS3 でサーボを制御する。 |
| ControlData | ESP-NOW で送受信する操作データ構造体。 |
| maxSpd | サーボ速度へのマッピング上限値。単位は SCServo ライブラリの速度値。 |
| ホイールモード | シリアルサーボを位置制御ではなく連続回転速度制御で使うモード。 |

## 7. 現在の設計方針

- ペアリング済み MAC アドレスではなくブロードキャストで送信し、受信側を固定しない。
- コントローラ側でジョイスティック補正・デッドゾーン処理まで行う。
- 車体側は受信済みの抽象操作量 `x`, `y` を単純な差動駆動式に変換する。
- 車体側のサーボ 4 個はすべて同じ速度指令系で制御する。
- 受信途絶時は停止するが、サーボへは毎ループ速度指令を出し続ける。
- 通信データ構造体・定数・MAC アドレス設定は `include/ControlProtocol.h` に集約する。
- 両デバイスは起動時に自分の STA MAC アドレスを画面表示し、その値を `ControlProtocol.h` の MAC 定義に設定できる。
